## 운영 원칙

- 번역과 포트폴리오를 분리한다.
- 작업 단위는 `페이지`가 아니라 `섹션`이다.
- 공개물은 번역량보다 이해, 구현, 응용을 보여준다.
- 모든 chapter는 동일한 QA 기준을 통과한 뒤 공개한다.

## 현재 초점

1. 16장 구조와 theme track를 manifest로 고정
2. chapter별 비공개/공개 템플릿 생성
3. GitHub Pages 배포 준비
4. 샘플 chapter 2개를 우선 완성

## 추천 첫 샘플

- Chapter 2. Lateral Vehicle Dynamics
- Chapter 6. Adaptive Cruise Control

이 두 chapter는 수식, 직관, 시각화, 시뮬레이션, 응용 포트폴리오를 모두 만들기 좋다.
