# Roadmap

## Phase 1. System Setup

- chapter manifest 확정
- glossary 정책 확정
- 사이트와 학습 자산 구조 분리
- GitHub Pages 배포 경로 준비

## Phase 2. Pilot Chapters

- Chapter 2와 Chapter 6을 샘플로 선정
- 번역, 5줄 요약, 식 정리, 구현 아이디어까지 완료
- 공개용 요약 페이지 2개 작성

## Phase 3. Track Expansion

- Track A와 Track B를 우선 확장
- 시뮬레이션 또는 그래프 산출물을 최소 1개씩 만든다
- glossary와 FAQ를 지속 보강

## Phase 4. Public Portfolio Polish

- 시각적 통일성 확보
- intro, about, learning rationale 정리
- 대표 결과물 중심으로 메인 페이지 다듬기
