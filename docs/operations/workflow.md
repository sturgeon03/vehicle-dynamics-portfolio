# Study Workflow

## 1. Intake

- chapter와 section 범위를 정한다.
- 페이지 범위와 관련 figure/table을 기록한다.
- glossary 충돌이 있는지 먼저 확인한다.

## 2. Translation

- 정밀 번역을 `study-assets/chapters/*/translation.md`에 쌓는다.
- 기호, 단위, 식 번호는 원문을 유지한다.
- 해설은 번역문과 분리한다.

## 3. Structuring

- 5줄 요약 작성
- 핵심 식 3~5개 선정
- 자주 헷갈리는 개념과 변수 정리

## 4. Implementation Hook

- Python/Matlab/Julia 중 하나로 재현 가능한 최소 산출물을 정의한다.
- 입력, 출력, 시나리오를 적는다.

## 5. QA

- 용어 일관성
- 식 번호 및 변수명 일치
- 공개용 페이지가 번역에만 머물지 않는지 점검

## 6. Publication

- 핵심 개념, 도식, 구현 링크, 관련 chapter 링크를 바탕으로 `docs/chapters/`를 다듬는다.
- 공개용 페이지는 원문 복사보다 설명과 응용 비중이 더 커야 한다.
